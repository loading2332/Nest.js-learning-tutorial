# 项目学习

- 我正在学习这个项目， 你制作的所有教程文件，都保存到 tutorial 文件夹当中。
- 我的疑问会放在 tutorial/QA 文件夹下，写入答案在 tutorial/QA 文件夹下的对应疑问文件中。
- 我的语法相关问题和笔记会放在 tutorial/syntax 文件夹下，写入或者完善答案在 tutorial/syntax 文件夹下的对应文件中。

# 关于syntax答案的写法

解释+示例代码

- 解释：对语法的解释，帮助理解语法的作用和用法，可能包括一些注意事项或者常见错误。
- 示例代码：提供一个或多个示例代码，展示语法的实际应用，并且注释清晰，帮助理解。

# 关于教程正文的写法

- 遇到新概念时，不能直接放代码。必须先解释“这是什么、为什么需要它、在当前项目里解决什么问题、什么时候应该用/不应该用”，解释时最好参考官方文档。再给代码。
- 代码块后要解释关键行的含义，尤其是第一次出现的 API、装饰器、配置项和数据库概念。
- 对复杂概念要先用业务场景讲清楚，再讲具体代码实现
- 不要假设我已经知道某个框架术语。第一次出现时要用定义解释，给出核心思维模型，再给专业名称。
