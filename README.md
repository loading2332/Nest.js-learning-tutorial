# NestJS 工程开发课程大纲

## 课程目标

本课程面向已经具备 TypeScript 和 Node.js 基础、希望能独立使用 NestJS 开发现代后端工程的学习者。完成课程后，你应该能够：

- 理解 NestJS 的核心设计：Module、Controller、Provider、依赖注入和装饰器。
- 独立搭建一个结构清晰、可测试、可维护的 NestJS API 项目。
- 使用 DTO、Pipe、Guard、Interceptor、Filter 等机制处理真实工程问题。
- 接入数据库、认证授权、配置管理、日志、测试和部署流程。
- 能够阅读 NestJS 官方文档并将新能力迁移到自己的项目中。

## 学习产出

课程最终会产出一个“课程管理 API”项目，包含：

- 用户注册、登录、JWT 认证。
- 用户、课程、章节、报名记录等资源管理。
- 参数校验、错误处理、分页查询和统一响应。
- 数据库持久化、配置管理、日志记录。
- 单元测试、端到端测试和基础部署说明。

## 推荐学习节奏

- 建议节数：18 节。
- 每节建议时长：60-120 分钟。
- 每 3-4 节完成一个阶段复盘。
- 每节包含三个部分：概念学习、代码练习、知识输出。

## 第一阶段：进入 NestJS 世界

核心内容：

- Express、Fastify、NestJS 的关系。
- 当前项目结构说明：`src/main.ts`、`app.module.ts`、`app.controller.ts`、`app.service.ts`。
- Nest CLI、`pnpm run start:dev`、`pnpm run test` 的基本使用。

代码练习：

- 新增一个 `/health` 接口，返回服务状态和当前时间。

- 画出一次 HTTP 请求从入口到返回响应经过的文件。

验收标准：

- `/health` 能返回 JSON。
- 能解释 `main.ts` 和 `AppModule` 的职责。

- 设计一组“课程章节”相关的 RESTful 路由。

验收标准：

- 能通过 HTTP 客户端调用课程相关接口。

### 第 3 节：Provider、Service 与依赖注入

### 第 4 节：Module 与工程结构

## 最终项目验收清单

- 项目结构清晰，业务模块边界明确。
- 所有输入参数都有 DTO 或 Pipe 处理。
- 常见错误能返回合适的 HTTP 状态码。
- 成功响应和错误响应格式稳定。
- 数据通过数据库持久化。
- 用户密码使用哈希保存。
- 受保护接口必须携带有效 JWT。
- 不同角色拥有不同接口权限。
- 核心 Service 有单元测试。
- 核心接口有端到端测试。
- 项目能通过 lint、test、e2e、build。
- README 包含启动、配置、测试、部署说明。

## 建议的学习方法

- 每节先完成最小可运行代码，再逐步优化结构。
- 遇到 NestJS 装饰器时，优先理解它解决的工程问题。
- 每学完一个机制，都问自己：它发生在请求生命周期的哪个位置。
- 不只写代码，也要输出文档、流程图或总结。
- 每个阶段都保留可运行版本，避免一次性堆太多改动。
